# Specs

Esta pasta contém especificações funcionais e jornadas de uso em Markdown.

Cada especificação pode originar:
- um ticket em `tickets/open/` quando o trabalho ainda precisa de refinamento;
- um execplan em `execplans/` quando o escopo já estiver claro e pronto para execução.

## Padrão oficial
- Guia completo: `SPECS.md`
- Template oficial: `docs/specs/templates/spec-template.md`

## Regras rápidas
- Toda spec deve ser documento vivo com status explícito (`draft`, `approved`, `in_progress`, `partially_attended`, `attended`, `superseded`).
- A spec deve manter rastreabilidade com tickets, execplans e commits relacionados.
- Uma spec só vai para `attended` com evidência observável de validação.
